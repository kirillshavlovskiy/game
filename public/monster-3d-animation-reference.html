Monster 3D — animation clip map

The repo does not include .glb files. When you add models under public/models/monsters/ and set NEXT_PUBLIC_MONSTER_3D=1, the combat portrait picks glTF animation clips by name using the tables below. Keep this file in sync with lib/monsterModels3d.ts when clip lists change.

Live / authoritative: With the app running, use the same data from source at /monster-3d-animations (e.g. http://localhost:3000/monster-3d-animations).

Portrait state → preferred clip names

StateCombat UI (summary)Clip names tried (first match wins)
idleCalm portrait; monster HP above ~⅓.Idle → idle → Rest → rest → Neutral → neutral
huntDefault threatening stance.Hunt → hunt → Walk → walk → Run → run → Aggro → aggro → Idle → idle
attackMonster attacking.Attack → attack → Strike → strike → Slash → slash
angryHard surprise stance.Hunt → hunt → Walk → walk → Run → run → Aggro → aggro → Idle → idle
rollingDice rolling (same clip list as attack).Attack → attack → Strike → strike → Slash → slash
hurtMonster took a hit.Hurt → hurt → Hit → hit → Damage → damage
defeatedMonster eliminated.Death → death → Defeated → defeated → Die → die → Down → down
recoverLow HP portrait band (~≤⅓).Recover → recover → Weak → weak → Tired → tired → Idle → idle
neutralSome UI fallbacks.Idle → idle → Rest → rest → Neutral → neutral
